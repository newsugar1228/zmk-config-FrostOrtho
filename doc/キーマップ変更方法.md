# キーマップの変更方法
Via等は使用できません。以下のいずれかの方法を実施ください。  

- [DYA Studio](https://studio.dya.cormoran.works/) で変更する　← おすすめ  
ブラウザ上で簡単にキーマップ・トラックボール速度等の変更ができ、ファームウェアの書き込み作業が不要です。  
スクロールの設定については対応できていませんmm  
iPadでもBluefyというブラウザで使用するとBluetooth接続で使用可能です。  
  - [DYA Studio を使用したキーマップ変更方法](#dya-studio-を使用したキーマップ変更方法)  

- [Keymap Editor](https://nickcoutsos.github.io/keymap-editor/) で変更する  
GitHubアカウントの作成が必要になり、変更するたびファームウェアの書き込みが必要になります。  
キーマップの変更ができますが、トラックボール速度等は変更できません。 
    - [Keymap Editor を使用したキーマップ変更方法](#keymap-editor-を使用したキーマップ変更方法)  
- ファームウェアを直接書き換える  
自由に変更や設定追加ができます。  
スクロールの設定やオートマウスレイヤーの挙動を変更したい場合はファームウェアを直接弄っていただく必要があります。  
    - [ファームウェアの書き換え](#ファームウェアの書き換え)  


## DYA Studio を使用したキーマップ変更方法
1. [DYA Studio](https://studio.dya.cormoran.works/)にアクセスして有線もしくはBluetooth接続を選択する。  
※ Bluetooth接続でFrostOrthoが検出されない場合、電源を一度オフにして再度オンにしてみてください。  
※ iPadはBluetooth接続のみ可能です。  
2. DYA Studioによる変更を許可するためにレイヤー4のstudio_unlockキーを押下する。  
![alt text](./img/dya-studio-2.png)  
3. キーマップやトラックボール設定の変更をして保存する。  
※ Battery Statusは現状対応していません。  

## Keymap Editor を使用したキーマップ変更方法

1. [ファームウェア用リポジトリ](https://github.com/imo00o/zmk-config-FrostOrtho)へアクセスする。  

2. 「fork」をクリックしてリポジトリをフォークする。  
![alt text](./img/image-4.png)  

3. 「Create fork」をクリックする。  
![alt text](./img/image-1.png)  

4. フォークができたら、Actions タブを開く。  

5. 「I understand my workflows, go ahead and enable them」をクリックし、GitHub Actionsを有効化する。  
![alt text](./img/image-3.png)  

6. [Keymap Editor](https://nickcoutsos.github.io/keymap-editor/)へアクセスする。  

7. GitHub を選択する。  
![alt text](./img/image-6.png)  

8. 「Login with GitHub」をクリックし、画面に従ってログインする。  

9.  「Authorize Keymap Editor」をクリックする。  
![alt text](./img/image-9.png)  

10.  「Add Repository」をクリックする。  
![alt text](./img/image-10.png)  

11.  「Only select repositories」を選択し、「Select repositories」からzmk-config-FrostOrthoを選択する。  
Installをクリックする。  
![alt text](./img/image-11.png)  

12.   キーマップの変更が終わったら、「Save」をクリックする。  

13.   「Latest」をクリックするとGitHub Actionsの画面へ遷移する。  
ビルドが完了するとファームウェアのダウンロードが可能となるのでダウンロードする。  
※ ビルドには2分程度かかる  
![alt text](./img/image-13.png)  


## ファームウェアの書き換え
 [Keymap Editor を使用したキーマップ変更方法](#keymap-editor-を使用したキーマップ変更方法)の1~5を実施したのち、自由にファイルの中身を変更ください。  
変更内容をpushするとGithubのActionsタブにファームウェアが出力されます。  

### カーソル移動量やスクロール量の設定変更
FrostOrtho_R.overlay を修正して調整してください。  

#### カーソル移動量（移動速度）
&zip_xy_scaler で設定できます。  

FrostOrtho_R.overlay
``` 
input-processors = <
    &mouse_runtime_input_processor
    &zip_temp_layer 6 100000 // AMLレイヤー、タイムアウト時間
    &zip_xy_scaler 2 3 // カーソル移動量2/3倍
    &zip_xy_transform INPUT_TRANSFORM_X_INVERT // カーソルの移動方向調整
    &zip_xy_transform INPUT_TRANSFORM_Y_INVERT // カーソルの移動方向調整
>;
```

#### スクロール量  
&zip_scroll_x_scaler、&zip_scroll_y_scalerで設定できます。  

FrostOrtho_R.overlay
``` 
&trackball_listener {
    scroller_y {
        layers = <5>; // 縦スクロールレイヤー
        input-processors = <
            // &scroll_runtime_input_processor
            &zip_scroll_transform (INPUT_TRANSFORM_X_INVERT) // スクロール方向調整
            &zip_xy_to_scroll_mapper
            &zip_scroll_x_scaler 0 3 // X軸スクロール量0/3倍
            &zip_scroll_y_scaler 2 3 // Y軸スクロール量2/3倍
        >;
    };
    scroller_x {
        layers = <7>; // 横スクロールレイヤー
        input-processors = <
            &zip_scroll_transform (INPUT_TRANSFORM_X_INVERT) // スクロール方向調整
            &zip_xy_to_scroll_mapper
            &zip_scroll_x_scaler 2 3 // X軸スクロール量2/3倍
            &zip_scroll_y_scaler 0 3 // Y軸スクロール量0/3倍
        >;
    };
};
```

### オートマウスレイヤーの設定変更

#### マウスボタンの変更
1. どのキーをマウスボタンとするか決める
2. AMLから離脱しないキーを新しいマウスボタンの位置に変更する  
FrostOrtho_R.overlayにて設定しています。こちらを変更してください。  
左上のQキーの位置を「0」として、左から右、上から下に数えた番号でキーを指定します。  

FrostOrtho_R.overlay
``` 
&zip_temp_layer {
    // キー入力後400ms以内はAMLに切り替わらない
    require-prior-idle-ms = <400>;
    // AMLから離脱しないキー
    excluded-positions = <
        6 // U 左クリック
        7 // I 右クリック
        8 // O 中央クリック
        20 // Z Shift
        30 // Control
        32 // Alt
    >;
};
```

3. キーマップを変更する（FrostOrtho.keymap のレイヤー6を変更）

#### AMLの誤爆対策
キー入力中に勝手にAMLに切り替わってしまった！みたいな事象がよく起こる場合はこちらを調整してみてください。

FrostOrtho_R.overlay  
``` 
&zip_temp_layer {
    // キー入力後400ms以内はAMLに切り替わらない
    require-prior-idle-ms = <400>;
```

#### オートマウスレイヤーをオフにする
下記の通りFrostOrtho.keymapとFrostOrtho_R.overlayを修正します。  
また、レイヤー1にマウスレイヤー(レイヤー6)に移動するキーを設定してください。

FrostOrtho.keymap
```
// AML クリックしたときにタイムアウトを延長
&mkp_input_listener { input-processors = <&zip_temp_layer 6 10000>; };
```

### エンコーダーの回転数変更
FrostOrtho.dtsi で変更できます。  
stepsとかtriggers-per-rotationを変更してみてください。

FrostOrtho.dtsi
```
left_encoder: encoder_left {
    compatible = "alps,ec11";
    a-gpios = <&xiao_d 5 (GPIO_ACTIVE_HIGH | GPIO_PULL_UP)>;
    b-gpios = <&xiao_d 0 (GPIO_ACTIVE_HIGH | GPIO_PULL_UP)>;
    steps = <24>;
    status = "disabled";
};
sensors: sensors {
    compatible = "zmk,keymap-sensors";
    sensors = <&left_encoder>;
    triggers-per-rotation = <10>;
};
```

## ファームウェアの書き込み方法
 [Keymap Editor を使用したキーマップ変更方法](#keymap-editor-を使用したキーマップ変更方法)や[ファームウェアの書き換え](#ファームウェアの書き換え)を実施してファームウェアをダウンロードし、解凍したら以下の手順でキーボードに書き込みます。  

1. 右手側キーボードとPCをケーブルで接続し、リセットスティックを2回押す

2. XIAO-SENSE(D:)のウィンドウが表示されるため、以下ファイルをドラッグ&ドロップして書き込む
    - FrostOrtho_R-seeeduino_xiao_ble-zmk.uf2

3. キーマップ以外の修正をした場合は、左手側キーボードも同じ手順で以下のファイルを書き込む
    - FrostOrtho_L-seeeduino_xiao_ble-zmk.uf2

4. 正常に書き込みできれば完了

### エラー発生時
書き込み中にエラーが発生した場合は、以下リセット用ファイルを同様の手順で書き込んでください。その後、もう一度ファームウェアの書き込みを実施してください。
- settings_reset-seeeduino_xiao_ble-zmk.uf2
